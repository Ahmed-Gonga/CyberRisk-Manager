# Security Controls

CyberRisk Manager tracks security controls and maps them to risks, owners, evidence, and basic framework references.

## Included Controls

## Multi-Factor Authentication

Reduces credential compromise and remote access risk.

Example mappings:

- ISO 27001: A.5.17
- NIST CSF: PR.AC

## Role-Based Access Control

Supports least privilege and access governance.

Example mappings:

- ISO 27001: A.5.15
- NIST CSF: PR.AC

## Security Logging and Monitoring

Supports detection, investigation, and incident response.

Example mappings:

- ISO 27001: A.8.15
- NIST CSF: DE.CM

## Secure Coding Review

Reduces application security issues such as injection flaws.

Example mappings:

- ISO 27001: A.8.28
- NIST CSF: PR.IP

## Backup and Recovery Procedure

Supports resilience and recovery readiness.

Example mappings:

- ISO 27001: A.8.13
- NIST CSF: RC.RP

## Evidence Tracking

Each control can include an evidence link to simulate audit evidence, such as access review records, backup test reports, code review results, or logging dashboards.
